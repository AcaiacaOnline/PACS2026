<analysis>**original_problem_statement:**
O usuário solicitou inicialmente um sistema de Plano Anual de Contratações (PAC). A aplicação foi expandida para incluir múltiplos módulos CRUD, gestão de usuários, dashboards e relatórios.

Nesta sessão, o usuário solicitou um novo conjunto de funcionalidades abrangentes:
1.  **Paginação Configurável:** Implementar paginação em todas as listas, permitindo que o usuário selecione a quantidade de itens por página (20, 30, 50, 100). Inicialmente implementado no frontend, foi posteriormente refatorado para ser controlado pelo backend.
2.  **Assinatura Digital Avançada:** Implementar um selo de assinatura visual em todos os PDFs gerados.
    *   O selo deve suportar assinaturas individuais e em lote (múltiplos assinantes).
    *   Deve conter um código de validação único e um QR Code.
    *   O CPF do usuário deve ser mascarado para conformidade com a LGPD.
    *   O layout do selo foi ajustado duas vezes: primeiro para ser mais compacto, e depois para ser movido para a lateral esquerda do documento.
3.  **Validação de Documentos:** Criar uma página pública para validar a autenticidade de qualquer documento gerado pelo sistema usando seu código de validação.
4.  **Ajustes de Cadastro de Usuário:** Adicionar campos obrigatórios ao perfil do usuário (Nome Completo, CPF, Cargo, Endereço, CEP) para serem usados na assinatura digital.
5.  **Histórico de Assinaturas:** Criar uma página onde cada usuário possa ver um histórico de todos os documentos que assinou.
6.  **Notificações por Email:** Enviar notificações por email aos usuários quando um documento que eles precisam assinar for publicado.

**PRODUCT REQUIREMENTS:**
-   **Máscaras de Input:** Todos os formulários do sistema devem aplicar máscaras automáticas para campos de moeda (R$), telefone, CEP, CPF, etc. (Tarefa herdada e concluída).
